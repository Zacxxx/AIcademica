# Repository Rules

## Ownership

- The repository is editable only by `zacxxx`.
- All changes must be attributed to `zacxxx` in the GitHub UI and in review policy.

## Enforcement Pattern

- Use `CODEOWNERS` to assign every path to `@zacxxx`.
- Require code owner review on protected branches.
- Do not merge changes that bypass owner approval.

## Editing Policy

- No direct edits by bots, assistants, or non-owner contributors.
- All pull requests must be approved by `zacxxx`.
- Any exception must be explicitly granted by `zacxxx`.

## Operational Note

- A repository file can document the rule.
- Actual enforcement still depends on GitHub branch protection and repository permissions.
